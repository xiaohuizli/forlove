# Reference Earth Particle Design

## Goal

把开场球体升级成参考图风格的宇宙发光地球：深空背景、蓝色海洋、绿色/金色大陆、白色云层和城市光点、蓝色大气辉光、多条绕地光轨。它仍然是粒子效果，不做图片贴图；用户挥手时，球体按照手的左右方向加速旋转。

## Visual Requirements

- 地球第一眼要能读出“海洋 + 大陆”，不能只是均匀噪点球。
- 海洋以深蓝、亮蓝、青蓝粒子为主，密度低于大陆，让球体有通透感。
- 大陆以绿色、青绿色、金色粒子为主，并按近似大陆轮廓聚集，密度明显高于海洋。
- 大陆边缘增加少量暖金色/白色城市光点，形成参考图里的发光细节。
- 云层使用白色/淡蓝白粒子，数量少，呈弧形云带，不遮住大陆。
- 外层保留大气辉光、星空背景、绕地光轨，但整体不要盖住地球主体。
- 粒子不应过于密集：大陆区域可以密，海洋区域要稀疏，星空和光轨作为辅助层。

## Interaction Requirements

- 默认进入页面后显示缓慢旋转的粒子地球。
- 手向左移动时，地球向左旋转并加速。
- 手向右移动时，地球向右旋转并加速。
- 手掌打开时，地球粒子散开；收拳时，粒子重新聚成地球。
- 严格流程仍然保持 `1 -> 2 -> 3 -> 4 -> 5/握拳`，不能回退或跳步。
- `4` 阶段继续显示 `I 爱心 YOU` 和烟花；`I`、爱心、`YOU` 的视觉间隔控制在约两个字符宽度。

## Implementation Approach

继续使用现有 Three.js 粒子 target 架构，不引入贴图或新渲染框架。核心改动集中在 `createEarthTarget`：

- 将地球粒子分成 ocean、land、coast/city、cloud 四类。
- 用投影平面的大陆中心点加多层噪声生成可读大陆块。
- 用拒绝采样控制密度：大陆最高，海岸次之，海洋最低，云层独立叠加。
- 导出或测试分类颜色比例，确保海洋蓝、大陆绿金、云层白都存在。

`EarthAtmosphereLayer` 只做环境辅助：星空、辉光和绕地轨迹。它不承担大陆/海洋表达，避免视觉上“热闹但不像地球”。

## Testing Plan

- 更新 `earthTarget.test.ts`，验证：
  - 地球边界仍保持球体尺寸。
  - 蓝色海洋、绿色/金色大陆、白色云层比例合理。
  - 大陆区域密度显著高于海洋区域。
  - 城市/海岸高光存在但比例受控。
- 运行完整 `npm test` 和 `npm run build`。
- 用 Playwright 截图检查 idle 地球是否接近参考图：星空、发光地球、稀疏海洋、密集大陆、光轨同时可见。
